How to make this site

May 6, 2017 hexo

hexo cheatsheet

install hexo with npm

https://coldnew.github.io/hexo-org-example/2017/03/05/getting-started-with-hexo-and-org-mode/ 

npm install -g hexo-cli

create a new blog / site

https://coldnew.github.io/hexo-org-example/2017/03/05/getting-started-with-hexo-and-org-mode/

This creates a new folder for the site, and then goes to that folder: 

hexo init my-blog && cd my-blog

Alternatively, if you do not name a folder, the site is installed in the current folder: 

mkdir my_new_blog
cd my_new_blog
hexo init

Install org renderer for hexo

https://coldnew.github.io/hexo-org-example/2017/03/05/getting-started-with-hexo-and-org-mode/ 

npm install https://github.com/coldnew/hexo-renderer-org#coldnew --save

config your site to use org-mode hexo renderer

https://coldnew.github.io/hexo-org-example/2017/03/05/getting-started-with-hexo-and-org-mode/

Add this to _config.yml: 

# hexo-renderer-org
## Docs: https://github.com/coldnew/hexo-render-org
org:
  emacs: /Applications/Emacs.app/Contents/MacOS/Emacs
  emacsclient: /Applications/Emacs.app/Contents/MacOS/bin/emacsclient
  cachedir: ./hexo-org-cache/
  theme: tango-dark
  common: |
    #+OPTIONS: html-postamble:nil

add htmlize highlighting (coloring) for code

https://coldnew.github.io/hexo-org-example/2017/03/05/getting-started-with-hexo-and-org-mode/

Add the following option to the above org: key-group (see previous section) 

org:
  # to make hexo-renderer-org htmlize content, this should set false
  htmlize: true

starting the server on a project

From inside the project's folder, run this: 

hexo server --draft --open

This starts the server with a few extra options:

  • –draft: Enable viwable draft posts (by default, drafts are hidden)
  • –open: Open the local site in your browser

A browser should pop up and display the default blog site with a canned "Hello World" post.

Install hexo-browsersync for automatic sync of edited pages

npm install hexo-browsersync --save
hexo server --draft --open # restart the server

generate the blog pages for publishing to web

hexo generate
# generates -> ./public/

Use the extra flag/option --watch to generate pages anew as they are edited: 

hexo generate --watch

deploy hexo pages on github-pages

https://gist.github.com/btfak/18938572f5df000ebe06fbd1872e4e39

Yaml configuration for site name + theming path

  • general site config: title, url, root etc
    # Site
title: yt8yt's note
subtitle:
description: yt8yt's personal blog
author: yt8yt
language:
timezone: Japan

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://yt8yt.github.io/
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

    NOTE: make sure to add the name of your repo to the root: key's path, if this is not your personal page but one that runs under a repo different than your users' name on github!

  • config git repo for deployment
    # Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:yt8yt/yt8yt.github.io.git
  branch: gh-pages

    NOTE: gh-pages is important!

install github deployer plugin

npm install hexo-deployer-git --save

deploying with deploy

hexo deploy

cleaning up published files in case of doubt

hexo clean

Removes published files. You may have to run: 

hexo generate

to re-generate the published files, before deploying.

how to add an "about" page and link it to the main menu create pages (instead of posts) (for linking to menu items)

http://stackoverflow.com/questions/29167023/how-to-add-route-for-hexo

answer 1

You have to create a new page by runnning hexo new page "about". A folder will be created in source/. In this folder, there is a file named index.md. In this file, you can write the content of this page /about in markdown.

This page will be displayed as a post page layout, if you want an other layout and style, define in front-matter, the layout with layout: YOURLAYOUT. Of course, you have to create this layout in the layout folder of your theme folder. The layout name will be the name of the file.

answer 2 (includes instructions for menu item in theme)

If you need a page about :

run hexo new page "about"

You will find an about folder in the source folder. You can edit the index.md file in the about folder.

add code About: /about in the menu section of _config.yml file in your theme folder. Preview your site :)

themes: lists and choices

hexo themes home page + gallery

https://hexo.io/themes/

current (DEFINITE!) first choice: hipaper

https://itimetraveler.github.io/hexo-theme-hipaper/ https://github.com/iTimeTraveler/hexo-theme-hipaper

Hiero is very similar, and an alternative, but I prefer the cleaner look of hipaper.

https://itimetraveler.github.io/hexo-theme-hiero/ https://github.com/iTimeTraveler/hexo-theme-hiero

  • clean look, white, no hero picture on top
  • resonsive, mobile compatible
  • simple sans font
  • great sidebar widget functionality: You can put your sidebar in left side, right side or bottom of your site by editing sidebar setting. Hiero provides 7 built-in widgets:
    1. search
    2. social
    3. recentposts
    4. category
    5. tag
    6. tagcloud
    7. archive
  • includes plugin for fast search, independent of third-party apps. Hiero use Insight Search to help you search anything inside your site without any third-party plugin.
  • includes plugin for fancybox to present picture galleries in various ways. Hiero uses Fancybox to showcase your photos. You can use Markdown syntax or fancybox tag plugin to add your photos.
  • disqus support for comments
  • Important: Contents are displayed in the sidebar! This enables long documents from org-mode. (!? to test…)

older first choice: blue lake

https://github.com/chaooo/hexo-theme-BlueLake/blob/master/README.en.md

  • responsive
  • mobile-ready
  • config example with menu for different pages/folders

perhaps even better? ylion/geekaholic

https://github.com/GeekaholicLin/hexo-theme-ylion

http://blog.geekaholic.cn/

another candidate: maupassant

https://github.com/tufu9441/maupassant-hexo
